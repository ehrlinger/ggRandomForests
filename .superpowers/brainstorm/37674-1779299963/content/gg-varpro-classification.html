gg_varpro — Classification: conditional = TRUE

How should conditional importance be shown for multi-class outcomes?

A — Stacked facet panels per class

facet_wrap(~class, ncol=1). Each class gets its own panel with shared variable axis. Colors shift per class. Cutoff line shown per panel. Most readable for 3+ classes with many variables.

B — Side-by-side panels per class

facet_wrap(~class, nrow=1). Classes span columns; variables share y-axis labels only in the leftmost panel. Better for 2–3 classes; can get cramped with many variables.

Context — conditional = TRUE

Applies only to classification forests. Uses the $conditional.z matrix from importance.varpro(local.std=FALSE) — one column per class. The unconditional z-scores (from the z vector) are shown when conditional = FALSE (default).
