gg_varpro — Default Plot Layout

How should variables be arranged in the default (non-faithful) view?

A — Horizontal boxplots (gg_vimp style)

Variables on y-axis, sorted by median z-score. Mirrors gg_vimp orientation. Red dashed line at cutoff. Familiar to ggRandomForests users.

B — Vertical boxplots

Variables on x-axis, boxes extend upward. More similar to base R bxp() convention. Natural for showing distribution height. Horizontal cutoff line.

Context

Both layouts show honest boxplots (hinges = 15th/85th percentile, whiskers = 5th/95th) rather than Tukey quartiles. The gg_vimp bar chart uses horizontal orientation — Option A would make gg_varpro feel like a natural companion. Option B matches varPro's own plot.varpro output more closely.
