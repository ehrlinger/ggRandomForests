# ggRandomForests v2.8.0 — Phase 0: Dependency Modernization & Migration Audit — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move `randomForestSRC` and `randomForest` from `Depends:` to `Imports:`, add `varPro` to `Imports:` and `igraph` to `Suggests:`, and repair every example/test that silently relied on those packages being *attached*, so `R CMD check` is 0 errors / 0 warnings / 0 notes on a clean base before any v2.8.0 component work begins.

**Architecture:** This is the prerequisite phase of the approved v2.8.0 varPro-integration design (spec: `~/Documents/ObsidianVault/R packages/ggRandomForests varPro Integration Design (2026-05-18).md`, §8 and §10). Components 1–3 (`gg_partial_varpro`, `gg_varpro`, `gg_udependent`) are planned separately and depend on this landing first and green. The package's R function bodies are already namespace-clean (NAMESPACE already `importFrom`s every cross-package symbol); the work here is the *attachment-removal fallout*: `@examples`, the `testthat` suite, and a regression guard that proves the migration actually took effect.

**Tech Stack:** R package tooling — `devtools`, `roxygen2`, `R CMD check`, `testthat` (3.x edition), `vdiffr`, `callr` (new Suggests, for clean-session regression test), `quarto` (vignette builder). Domain packages: `randomForestSRC` (>= 3.4.0), `randomForest`, `survival`, `varPro` (CRAN), `igraph`.

**Critical domain note — the `Surv()` footgun.** `randomForestSRC::rfsrc(Surv(time, status) ~ ., data = ...)` evaluates the formula in the *caller's* environment. `Surv` resolves today only because `Depends: randomForestSRC` attaches `randomForestSRC`, whose own `Depends: survival` puts `Surv` on the search path. After Phase 0, `library(ggRandomForests)` no longer attaches `randomForestSRC`, so any example/test that writes a bare `Surv(...)` inside an `rfsrc()` formula **will error with `could not find function "Surv"`** unless that code attaches `survival` itself or the symbol is otherwise resolvable. This is the single highest-risk item; Task 7 contains an empirical probe to fix the pattern before bulk-applying it. Do not guess the pattern — run the probe.

The dev series base is **last-released + `.9000`** = `2.7.3.9000` (R-pkgs convention; `2.8.0.9000` is rejected because `numeric_version` orders it *above* the eventual `2.8.0` CRAN release, which would break the upgrade path for dev installs). ## Task 4: Regression Guard — Namespace Hygiene Test (write it failing)

**Files:**
- Create: `tests/testthat/test_namespace_hygiene.R`

This test encodes the migration *invariant* so it cannot silently regress later: (a) `DESCRIPTION` `Depends:` contains only the R version constraint, and (b) ggRandomForests' core extractors work in a clean R session where `randomForestSRC`/`randomForest`/`survival` are loaded *namespaced but not attached*. It is written now so it **fails** before the DESCRIPTION change, then drives Tasks 5–8.

- [ ] **Step 1: Write the failing test**

Create `tests/testthat/test_namespace_hygiene.R`:
```r
# Regression guard for the v2.8.0 Phase 0 dependency modernization.
# Asserts the Depends->Imports migration took effect and stays effective.

test_that("DESCRIPTION Depends declares only the R version constraint", {
  deps <- read.dcf(testthat::test_path("..", "..", "DESCRIPTION"),
                    fields = "Depends")[1, 1]
  deps_pkgs <- trimws(strsplit(deps, ",")[[1]])
  # Every entry must be the R(>= x) constraint, not a package.
  # NB: `\b` is a backspace in R's default TRE regex, not a word
  # boundary — use an explicit whitespace/end-of-string class.
  expect_true(all(grepl("^R([[:space:]]|$)", deps_pkgs)),
              info = paste("Depends still attaches packages:", deps))
})

test_that("core extractors run without randomForestSRC/randomForest attached", {
  skip_if_not_installed("callr")
  skip_if_not_installed("randomForestSRC")
  skip_if_not_installed("randomForest")
  skip_if_not_installed("ggRandomForests")

  out <- callr::r(function() {
    # Load ggRandomForests WITHOUT attaching its former Depends.
    suppressMessages(library(ggRandomForests))
    attached <- search()
    stopifnot(!("package:randomForestSRC" %in% attached))
    stopifnot(!("package:randomForest" %in% attached))

    # A regression forest extractor must work with rfsrc only namespaced.
    rf <- randomForestSRC::rfsrc(Ozone ~ ., data = airquality,
                                 ntree = 30, na.action = "na.impute")
    g <- gg_error(rf)
    inherits(g, "gg_error")
  })
  expect_true(out)
})
```
